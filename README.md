# AI-Fund-Lab-1000

一个以初始资金 1000 元运行的公募基金决策实验，用于长期记录每日判断、真实执行状态和组合绩效。

> 本仓库记录研究与实验过程，不构成收益保证。真实交易始终由用户决定并执行。

## 决策归档结构

- `decisions/YYYY/MM/README.md`：当月决策索引和阶段统计。
- `decisions/YYYY/MM/YYYY-MM-DD.md`：单个交易日的完整决策。
- 年份和月份均使用数字，月份固定为两位数，便于长期排序和程序读取。

## 每日记录固定结构

1. 今日结论
2. 账户状态
3. 市场环境
4. 当前组合分析
5. 今日操作步骤
6. 不选择其他方案的原因
7. 黑天鹅与反证条件
8. 下一交易日关注
9. 数据来源与完整性
10. 执行状态

每日文件顶部同时保留结构化元数据，包括日期、决策类型、建议金额、现金、持仓数量、单位净值和执行状态，方便未来自动汇总。

## 执行状态定义

- `proposed`：已提出建议，等待用户决定。
- `pending_confirmation`：用户已提交订单，但份额或成交净值尚未确认。
- `confirmed`：用户已反馈并确认实际成交。
- `not_executed`：建议未执行。
- `no_trade`：当日正式结论是不交易。

没有用户反馈时，不得把建议写成实际成交。

## 最高优先级规则

- 初始资金为 1000 元；追加资金每次 1000 元，必须由用户批准。
- 最多同时持有 5 只基金，绝不允许出现第 6 只。
- 0只基金和100%现金均为合法状态。
- 非实验旧仓不进入本实验的资金、持仓、收益和回撤核算。
- 每个发起讨论的基金交易日都要形成明确判断，但不强制交易。
- 用户负责真实申购、赎回和转换；AI只负责研究、建议和记录。
