# AI-Fund-Lab-1000

一个以初始资金1000元运行的AI公募基金决策实验，用于长期记录每日判断、真实执行、费用、复盘和组合绩效。

> 项目以实际盈利为目标，但不承诺收益。AI负责研究、选基、仓位和操作建议，用户负责京东金融中的真实执行与最终批准。

## 决策归档结构

- `decisions/YYYY/MM/README.md`：当月决策索引和阶段统计。
- `decisions/YYYY/MM/YYYY-MM-DD.md`：单个交易日的完整决策。
- 年份和月份均使用数字，月份固定为两位数。

本地养基宝、京东金融和外部信息截图保存在私人工作目录的 `evidence` 中，不上传本公开仓库。

## 核心规则

- 同时持有0—5只基金；5只是绝对上限，绝不出现第6只。
- 整体仓位可为0%—100%，不为填满仓位乱买基金。
- 单只基金首次买入原则上不超过总资产30%，累计原则上不超过50%。
- 单基金从首次份额确认日开始，在第10、15、30个自然日复盘。
- 单基金30天主要目标为扣除预计交易费用后净收益5%，但不构成保证。
- 国内基金、黄金、债券和QDII均可研究；实际可购和费用以京东金融为准。
- 养基宝用于盘中观察，正式净值和交易以基金公司及京东确认结果为准。
- 个人旧仓完全排除在实验资金、持仓、收益和回撤之外。
- 所有数据必须真实，成功和失败都要保留。
- 只有用户明确说“上传到GitHub”后才更新本仓库。

## 执行状态

- `proposed`：AI已提出，等待用户阅读。
- `approved_pending_execution`：用户认可，尚未提交。
- `pending_confirmation`：已提交，等待份额或净值确认。
- `confirmed`：真实成交已确认。
- `not_executed`：建议没有执行。
- `no_trade`：当日正式结论是不交易。
- `no_session`：用户当天没有发起决策会话。

## 历史真实性

决策上传后不得根据后来行情重写原始分析。正式净值、成交结果、错误修正和复盘只能以追加说明方式记录，并注明时间和依据。
