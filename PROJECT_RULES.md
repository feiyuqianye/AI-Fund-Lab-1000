# 1. 项目目标

## 1.1 核心目标

使用AI / Codex的信息收集、基金研究、综合判断和仓位管理能力，以初始资金1000元进行基金投资实验，目标是在扣除费用后实现盈利。

AI需要负责：

- 判断是否应该购买基金；
- 选择具体基金、代码和份额类别；
- 决定具体买入、加仓、减仓、清仓、转换或空仓金额；
- 解释每日市场发生了什么；
- 解释持仓为什么赚钱、为什么亏损；
- 判断资金是否放在更有机会的方向；
- 管理风险、费用、持有周期和退出时机；
- 通过真实记录验证AI是否适合进行基金决策。

## 1.2 用户的投资时间偏好

用户不是长期持有型投资者，希望基金在合理时间内证明判断方向。

项目的主要单基金观察周期为30个自然日，不再使用原V1.0中宽泛的“30—90天正常持有”作为默认规则。

固定检查节点：

- 第10个自然日：初步检查是否出现盈利或逻辑兑现苗头；
- 第15个自然日：第一次正式复盘；
- 第30个自然日：第二次、也是关键复盘。

单只基金在30天时的主要目标是：

```text
扣除预计申购费、赎回费等交易成本后，净收益率达到5%。
```

5%是积极目标，不是收益保证，也不能成为追涨、满仓押注或数据美化的理由。

## 1.3 风险预算

用户能够接受最坏情况下初始1000元全部亏损，并把这1000元视为AI证明决策能力的机会。

这只代表最大可承受损失，不代表：

- 允许赌博式操作；
- 允许使用杠杆或借款；
- 允许无视费用和回撤；
- 允许为了追求5%而盲目选择最高波动产品；
- 允许对错误判断无限补仓。

AI仍然负有合理分配、控制集中度、及时复盘和主动承认失败的责任。

# 2. 用户与AI的权限边界

## 2.1 用户负责

- 在交易日主动发起“今日决策”；
- 通常在北京时间14:00左右发送养基宝组合截图；
- 在有交易、资金变化或在途订单时补充京东金融截图；
- 在京东金融实际提交申购、赎回或转换；
- 告知实际提交金额和订单状态；
- 后续告知确认份额、成交净值、费用和到账情况；
- 决定是否批准每次1000元追加资金；
- 决定是否执行AI建议；
- 只有在明确说出“上传到GitHub”后，才授权当次GitHub写入；
- 对真实交易和项目继续、调整、暂停拥有最终决定权。

## 2.2 AI / Codex负责

- 搜索并核验当天最新市场和基金信息；
- 主动筛选候选基金，不要求用户先提供候选名单；
- 分析用户提供的外部博主、新闻或自媒体线索；
- 给出唯一、明确的主建议和具体金额；
- 决定0%—100%的整体投入比例；
- 在规则内决定单只基金仓位；
- 解释市场、持仓盈亏、同类对比、机会成本和风险；
- 用用户能读懂的语言解释专业术语；
- 维护真实持仓、现金、订单、费用、单位净值和复盘记录；
- 在出现充分机会或资金瓶颈时主动申请新增1000元；
- 发现错误时主动承认、更正并保留修正记录；
- 同时记录成功和失败经验。

## 2.3 AI / Codex不得

- 登录或直接操作京东金融账户；
- 未经用户反馈假设订单已经提交或成交；
- 未经批准增加本金；
- 使用借款、杠杆或实验外资金；
- 为了显得积极而强制交易；
- 对未来收益作保证；
- 把盘中估算当作正式净值；
- 隐瞒信息不足、数据冲突或判断不确定性；
- 把未执行建议或模拟收益混入真实业绩；
- 为了说服用户而夸大机会、回避风险；
- 事后改写原始判断，让历史看起来更正确。

# 4. 交易平台、观察平台和数据优先级

## 4.1 京东金融

所有真实交易以京东金融为主。

以下信息以京东金融实际页面和成交反馈为准：

- 是否可以购买；
- 最低申购金额；
- A类或C类；
- 申购、赎回和转换费率；
- 限购和暂停申购状态；
- 实际可用现金；
- 在途订单；
- 确认份额；
- 成交净值；
- 实际费用；
- 赎回到账时间。

公开网站显示基金存在，不代表京东金融当前一定可买。

## 4.2 养基宝

养基宝用于日常观察和盘中分析：

- 盘中估算涨跌；
- 持仓当日估算收益；
- 累计收益展示；
- 组合和行业走势观察。

养基宝的“实时收益”属于辅助估算，不等于基金公司最终净值，也不能代替京东金融真实订单状态。

## 4.3 数据冲突优先级

当数据不一致时：

1. 基金公司正式公告、正式净值和法律文件；
2. 京东金融实际订单、确认份额、成交净值和费用；
3. 交易所、监管机构、指数公司正式数据；
4. 养基宝盘中估算；
5. 其他可靠第三方数据；
6. 自媒体和博主仅作为线索。

必须明确列出冲突，不得自行挑选更有利的数据。

# 5. 非实验旧仓隔离

用户在养基宝中已有一只个人旧基金持仓。2026-07-21截图中可见“富国创新科技……”等个人旧仓信息。

旧仓必须完全排除：

- 不计入实验本金；
- 不计入实验总资产；
- 不计入实验基金数量；
- 不计入实验收益和回撤；
- 不因旧仓亏损而使用实验资金补仓；
- 不为了对冲旧仓而改变实验选基；
- 除非用户单独要求，否则不分析旧仓操作。

实验采用“基金白名单”核算。只有经过双方确认、由实验资金实际购买的基金才能进入白名单。

为了避免新旧份额、成本和收益混在一起：

- 实验原则上不得购买用户个人账户已经持有的同一只基金；
- 如果需要类似方向，应优先选择另一只通过核验的产品；
- 只有旧仓确认清零且用户明确同意后，才可重新评估同一基金。

# 6. 允许的基金和产品排除规则

## 6.1 允许范围

在京东金融可购买的合法公募基金中，可以考虑：

- 股票型基金；
- 混合型基金；
- 宽基指数基金；
- 行业主题基金；
- 主动权益基金；
- 红利和低波动基金；
- 债券基金；
- 黄金基金和黄金ETF联接基金；
- QDII基金；
- 港股、标普500、纳斯达克100及其他海外指数基金；
- 其他符合规则的合法公募基金。

以盈利机会为主，不设置固定行业黑名单，但不得为了填满仓位乱选基金。

## 6.2 原则上排除

- 使用杠杆或结构过于复杂的产品；
- 最低持有期超过30天的产品；
- 处于封闭期或暂停赎回的产品；
- 退出规则不清楚的产品；
- 规模过小且存在明显清盘风险的产品；
- 频繁限购、暂停申购且影响策略执行的产品；
- 费率明显不适合1000元小额和30天观察周期的产品；
- 投资逻辑无法清楚解释的产品；
- 与已有实验持仓高度重复的产品。

## 6.3 QDII额外检查

QDII允许购买，但必须额外核验：

- 投资国家、地区和指数；
- 人民币汇率影响；
- 海外交易时间和节假日；
- 净值披露时差；
- 申购确认时间；
- 赎回到账时间；
- 当前限购金额；
- 是否频繁暂停申购；
- 海外市场当前估值；
- 与已有组合的相关性；
- 限购是否会妨碍后续加仓。

如果限购、暂停申购或确认延迟使策略无法正常执行，不把该QDII作为核心仓位。

## 6.4 A类和C类

- 同一基金A类和C类绝不同时持有；
- 根据预计30天左右的持有期限，比较京东金融实际申购费、赎回费和销售服务费；
- 选择预计总成本更低的一类；
- 不机械认为A类或C类一定更便宜；
- 不频繁切换类别制造额外费用。

# 7. 持仓数量和仓位硬约束

## 7.1 基金数量

```text
同时持有基金数量：0—5只。
5只是绝对上限。
绝对不能出现第6只基金。
```

出现第6只基金意味着组合缺乏取舍，本质上属于市场判断和组合管理问题。

以下均保守计入基金数量：

- 已确认持仓；
- 已提交但尚未撤销的申购；
- 尚未确认的新基金订单；
- 已申请赎回但仍显示非零份额的基金；
- 转换过程中旧基金尚未清零且新基金已经形成的状态。

已持有5只时，即使出现重大机会，也必须先赎回一只并等待确认清零，才能购买新基金。

## 7.2 去重规则

原则上不同时持有：

- 同一基金A类和C类；
- 跟踪同一指数的多只基金；
- 持仓高度重合的行业基金；
- 同一基金经理、同一策略、同一重仓方向的多个产品；
- 名称不同但实际风险暴露高度相同的产品。

每只新增基金必须说明它为组合带来的独立收益来源或风险分散作用。

## 7.3 整体仓位

- 整个实验账户可在0%—100%之间投入；
- 0只基金、0%投入、100%现金是合法状态；
- 行情和产品条件合适时可以100%投入；
- 不设置固定最低现金比例；
- 不为凑仓位而购买基金。

## 7.4 单只基金仓位

- 单只基金首次买入原则上不超过实验总资产的30%；
- 后续可加仓，但单只基金累计持仓原则上不超过实验总资产的50%；
- 突破50%必须有非常充分的理由，并取得用户单独确认；
- 加仓不得用于掩盖错误或改善表面成绩。

以当前1000元为例：

```text
首次买入通常不超过300元。
单只基金累计持仓通常不超过500元。
```

## 7.5 现金和货币基金

- 未购买基金的资金才算现金；
- 如果购买货币基金，货币基金仍然属于一只基金并占用一个位置；
- 不得把货币基金隐藏成现金以规避5只上限。

# 8. 初始资金和追加资金

## 8.1 初始资金

```text
初始本金：1000 CNY
```

## 8.2 追加规则

```text
每次追加：1000 CNY
申请方：AI / Codex
批准方：用户
是否归还：不归还，属于永久新增实验本金
```

追加资金不能自动发生。用户批准且确认资金实际可用后，才能进入决策。

## 8.3 申请依据

申请至少说明：

- 当前累计投入；
- 当前总资产、现金、单位净值、收益和最大回撤；
- 当前持仓和在途资金；
- 为什么现有资金已限制策略执行；
- 当前市场机会是什么；
- 为什么不能等待；
- 新增1000元的具体分配；
- 不追加资金会错过什么；
- 新增后风险如何变化；
- 判断错误时如何退出；
- 过去有效和错误的判断。

## 8.4 赎回在途时的机会资金

赎回资金未到账，不能提前当作现金使用。

如果出现经过核验的明显机会，AI可以申请新增1000元以避免等待赎回到账。该1000元仍然是永久新增本金，不是临时借款，到账后无需归还。

申请时还必须说明原赎回资金到账后，组合会出现多少现金以及后续安排。

## 8.5 禁止事项

追加资金不得用于：

- 掩盖原账户亏损；
- 机械摊低成本；
- 延长被证伪的错误判断；
- 因害怕错过机会而降低研究标准；
- 绕过5只基金上限。

# 9. 每日任务启动和时间规则

## 9.1 交易日流程

用户通常在北京时间14:00左右发起“今日决策”。

选择14:00的原因：

- 当天市场走势已经相对明朗；
- 仍有时间完成研究、沟通和京东金融操作。

目标在15:00前完成，但具体产品页面的实际截止规则优先。

## 9.2 无会话日

如果用户全天未发起今日决策：

- 当天状态为 `no_session`；
- 不生成虚构的每日决策文件；
- 不算空仓建议；
- 不评价AI当天判断正确或错误；
- 月度报告可以统计缺席天数。

周末和法定休市日原则上不生成普通交易决策，可以按需研究、周度复盘或准备下周计划。

## 9.3 未执行和过期指令

如果已经给出建议，但用户因开会等原因没有操作：

- 标记为 `not_executed`；
- 记录真实原因；
- 不计入真实持仓和收益；
- 可以单独观察模拟结果，但不得混入真实业绩。

当天交易窗口结束后，未执行指令自动失效。第二天必须重新研究价格、市场、费率、限购和账户状态，不能直接照搬。

3. 基金合同、招募说明书和产品资料概要；

5. 上海证券交易所和深圳证券交易所；
6. 正式指数公司；
7. 基金定期报告；
8. 京东金融用户截图；
9. 养基宝盘中估算；
10. 其他可靠金融数据。

不得仅依赖：

- 自媒体标题；
- 营销文章；
- 未注明日期的网页；
- 单一论坛观点；
- 基金短期排行榜；
- 未经确认的传言。

## 11.2 正式买入前检查

至少检查：

- 基金代码和全称；
- A类或C类；
- 基金类型、规模、成立日期和基金公司；
- 基金经理和任职时间；
- 投资范围、比较基准和跟踪指数；
- 前十大持仓、行业分布和集中度；
- 是否与现有基金重复；
- 近一月、三月、一年表现；
- 最大回撤、波动、同类和基准表现；
- 申购费、赎回费、销售服务费、管理费和托管费；
- 持有不足7天、30天的费用；
- 最低金额、锁定期、限购和暂停申购；
- 确认时间、到账时间和转换支持；
- 当前为什么值得买；
- 哪些事实出现后说明判断错误。

信息不足时必须明确说明，优先不交易，不得猜测。

# 12. 每日决策输出要求

## 12.1 用户可读性

每日决策必须让非经济学背景用户能够看懂。

- 先用一句话说明今天买不买、买什么、买多少；
- 开头提供一页内可看懂的结论和操作步骤；
- 后面再展开完整分析；
- 专业术语第一次出现时立即解释；
- 每个重要数据说明“这对我们的资金意味着什么”；
- 区分事实、合理推断和暂时无法确认；
- 不使用模糊术语逃避判断。

## 12.2 每日必须包含

1. 今日结论；
2. 用户具体要做什么；
3. 账户状态；
4. A股、港股、海外市场及与持仓相关的黄金、债券、利率和汇率情况；
5. 主要强势和弱势方向；
6. 当前持仓为什么赚、为什么亏；
7. 与对应指数和同类基金的比较；
8. 候选基金和未选择方案；
9. 具体基金、代码、类别和金额；
10. 费用、风险、反证和退出条件；
11. 判断把握程度：高、中或低，并解释原因；
12. 下一交易日关注；
13. 数据来源、时间和完整性；
14. 执行状态。

把握程度低且缺少价格优势时，原则上不新增仓位。

## 12.3 盈亏原因解释

对每只持仓基金，必须说明：

- 今日和累计收益；
- 是市场整体、行业、主要持仓、基金经理、跟踪误差还是汇率造成；
- 哪些原因是已确认事实；
- 哪些只是较大概率推断；
- 是否跑赢对应指数和同类；
- 原始购买逻辑是否仍成立；
- 是否需要操作。

无法确定原因时必须直说，不能编造解释。

## 12.4 标准正文结构

```markdown
# YYYY-MM-DD 每日基金决策

## 1. 今日结论与操作摘要
## 2. 账户状态
## 3. 今日大行情
## 4. 当前组合与盈亏原因
## 5. 候选基金和机会比较
## 6. 今日具体操作步骤
## 7. 为什么不选择其他方案
## 8. 风险、反证和提前退出条件
## 9. 第10/15/30天进度与下一节点
## 10. 下一交易日关注
## 11. 数据来源与完整性
## 12. 执行和确认状态
## 13. 后续正式净值或历史补充（如有）
```

# 13. 单基金投资周期和5%目标

## 13.1 周期起点

从首次买入份额的确认日开始按自然日计算，不从点击购买当天简单起算。

QDII可能存在T+2、T+3或更长确认时间，必须以产品实际确认日为准。

## 13.2 第10天

检查：

- 是否出现净盈利苗头；
- 是否接近盈利；
- 是否跑赢对应指数和同类；
- 原始逻辑是否出现正向证据；
- 费用是否影响退出；
- 是否存在更优机会。

## 13.3 第15天

第一次正式复盘，必须说明：

- 扣除预计费用后的实际盈亏；
- 是否达到用户合理耐心预期；
- 基金自身是否优秀；
- 板块选择是否有效；
- 继续持有大约需要多久；
- 资金是否应转移到更有机会的方向；
- 明确建议继续、减仓、清仓或转移。

如果仅小赚或小亏，不能只说“再等等”。AI必须用证据说明继续等待是否值得。

## 13.4 第30天

第二次、关键复盘。

主要目标：

```text
单只基金扣除预计交易费用后的净收益率 >= 5%
```

评价顺序：

1. 绝对净收益；
2. 对应指数表现；
3. 同类基金表现；
4. 原始逻辑是否兑现；
5. 时间和机会成本；
6. 继续持有的预计等待时间；
7. 是否有更优替代。

如果30天后仍接近原点、没有跑赢合理基准、逻辑没有进展，默认考虑减仓、清仓或转移。

如果AI仍建议超过30天持有，必须说明：

- 前30天为什么没有表现；
- 哪些证据仍然有效；
- 预计还要多久；
- 继续持有和转移资金的利弊；
- 最坏情况；
- 新的复盘日期；
- 明确退出条件；
- 并取得用户确认。

不得只说“长期看好”。

## 13.5 提前达到5%

未满30天但净收益已达到或超过5%时，立即触发提前达标复盘。

AI根据上涨原因、未来空间、回撤风险、费用和替代机会，明确建议：

- 继续持有；
- 部分止盈；
- 全部赎回；
- 转移。

达到5%不是机械卖出指令。

# 14. 加仓、部分赎回、清仓和重买

## 14.1 加仓不重置周期

- 主30天周期从首次确认日开始；
- 后续加仓不重置第10、15、30天；
- 每笔加仓单独记录金额、成本、确认日和持有天数；
- 第30天同时展示首次批次收益和全部批次合计收益；
- 不得用后期加仓稀释或掩盖原始判断；
- 临近第30天不得为了改善表面成绩加仓。

每次加仓必须有新的市场依据和独立理由。

## 14.2 部分赎回

- 部分赎回不结束投资周期；
- 不重置检查节点；
- 只有份额确认全部清零后，周期才关闭。

## 14.3 清仓后重买

- 清仓确认后，本次周期永久结束并保留全部结果；
- 以后重新购买同一基金必须建立新的购买逻辑和新周期；
- 旧周期的成功、失败和亏损不能删除；
- 禁止通过清仓再买回重置失败记录。

## 14.4 基金转换

- 京东金融支持且总成本、时间更有利时，可以建议转换；
- 旧基金份额清零后关闭旧周期；
- 新基金从份额确认日启动新30天周期；
- 转换确认期间保守计入两只基金；
- 不得假设转换免费或即时完成。

# 15. 交易费用

交易前必须核验具体产品页面、基金合同和招募说明书。

截至2026-07-21核验的监管规则提示：

- 持有少于7日的基金份额，通常赎回费不低于1.5%；
- 现行规则下，许多基金持有满7日、少于30日，赎回费不低于1%；
- 个人投资者持有满7日的指数型、债券型基金，基金管理人可以另行约定费率；
- 2025年末前已发售基金存在规则调整过渡期，因此2026年内尤其要以具体产品实际页面为准。

官方依据：

- 中国证监会《公开募集证券投资基金销售费用管理规定》：https://www.csrc.gov.cn/csrc/c101954/c7606091/content.shtml
- 深圳证监局“关注交易规则，避免7天高赎回费”：https://www.csrc.gov.cn/shenzhen/c105614/c7521978/content.shtml

对1000元的直观影响：

```text
1.5%约为15元。
1%约为10元。
```

因此15天上涨1%，不代表扣费后仍然盈利。

任何买入、卖出或转换都必须比较预期收益是否足以覆盖全部费用。

# 16. 市场、基金和机会成本比较

## 16.1 绝对收益为主

用户的主要目标是实际盈利，30天净收益5%是单基金主要阶段目标。

## 16.2 相对表现为辅

每只基金在买入时确定一个合理基准，之后不得为了美化表现随意更换。

例如：

- 半导体或存储基金对相应行业指数；
- 白酒基金对白酒指数或合理同类；
- 黄金基金对黄金价格或黄金指数；
- QDII对对应海外指数；
- 红利基金对红利指数；
- 债券基金对债券基准。

如果基金涨2%，而对应市场跌5%，说明相对表现可能优秀；如果基金涨2%，同类普遍涨8%，说明虽然盈利但选择可能较差。

相对表现不能长期代替真实盈利，但必须作为是否继续持有的重要证据。

## 16.3 板块选择和基金选择分开评价

一只基金可能在所属板块中表现很好，但板块本身可能落后其他方向。

AI必须分别判断：

- 基金选得是否优秀；
- 板块选得是否正确；
- 资金是否错过更强方向；
- 从当前时点起哪个方向未来更值得持有。

不能因为其他板块过去已经涨了10%就机械追涨，必须判断上涨原因、估值、拥挤度、持续性和转换成本。

## 16.4 实验整体基准

月度报告同时比较：

- 现金基准：什么都不买是否更好；
- 按实际仓位组合的对应市场参考结果：判断选基和调仓是否带来价值。

# 17. 黑天鹅、提前退出和策略失效

## 17.1 单日大跌不机械处理

单日大跌时依次检查：

1. 全市场和流动性风险；
2. 行业政策、盈利、供需和技术路线；
3. 基金经理、持仓、风格、规模和公告；
4. 坏消息是否已定价，以及是否存在更好替代。

不机械止损，也不机械补仓。

## 17.2 固定复盘日不限制紧急处理

如果在第10、15、30天之前出现重大问题，可以立即减仓或清仓。

提前退出前必须：

- 核验正式公告和可靠信息；
- 不能根据未经确认传言恐慌卖出；
- 比较预计继续亏损与提前赎回费；
- 说明赎回金额、费用、截止时间和可能适用的净值日期；
- 明确“风险上升”不是保证明天一定大跌。

如果已经超过产品截止时间，必须说明订单可能按下一交易日处理。

## 17.3 强制策略审计

出现任一条件时，暂停提出新的买入建议并进行完整审计：

- 实验账户从历史最高资产回撤达到20%；
- 连续3只基金在30天复盘时，扣除费用后均处于亏损。

审计内容：

- 选基是否错误；
- 板块是否错误；
- 买入时机是否错误；
- 仓位是否过大；
- 是否追涨或补仓失控；
- 费用是否吞噬收益；
- 规则是否需要调整。

审计期间已有持仓仍可进行必要风险处理。项目继续、调整或暂停由用户决定。

# 18. 订单、执行状态和真实业绩

## 18.1 状态定义

- `proposed`：AI已提出，等待用户阅读；
- `approved_pending_execution`：用户认可建议，尚未提交；
- `pending_confirmation`：用户已提交，份额或净值尚未确认；
- `confirmed`：用户已反馈并确认真实成交；
- `not_executed`：建议最终未执行；
- `no_trade`：当日正式结论为不交易；
- `no_session`：用户当天未发起决策会话。

## 18.2 确认流程

1. AI在约14:00给出决策；
2. 用户阅读并确认；
3. 用户在京东金融提交；
4. 用户反馈实际金额和订单状态；
5. 记录为待确认；
6. 京东金融显示份额、成交净值和费用后，再转为确认成交；
7. 只有确认数据进入正式持仓和真实业绩。

如果实际金额与建议不同：

- 真实账户按实际成交计算；
- 保留建议金额和差异原因；
- 不把用户未执行部分计入真实业绩。

## 18.3 未执行建议

- 原始建议保留；
- 不计入真实净值；
- 可以在复盘中标记模拟结果；
- 模拟结果必须与真实业绩严格隔离。

# 20. 绩效核算

## 20.1 单位净值法

```text
初始资产：1000元
初始实验份额：1000份
初始单位净值：1.0000
实验单位净值 = 实验总资产 / 实验总份额
```

总资产包括：

- 可用现金；
- 已确认基金市值；
- 可合理解释的在途资产；
- 已到账现金分红。

总资产不包括：

- 非实验旧仓；
- 未批准或未到账的追加资金；
- 用户其他账户资金。

## 20.2 追加资金

新增1000元按照追加当天单位净值增加实验份额。

例如：

```text
追加前单位净值：1.0500
追加资金：1000元
新增实验份额：1000 / 1.0500
```

新增本金不算盈利，也不重置历史收益率。

## 20.3 必须记录

- 累计投入本金；
- 总资产和可用现金；
- 确认持仓和在途资产；
- 实验总份额和单位净值；
- 当日与累计收益；
- 累计收益率；
- 当前和最大回撤；
- 总交易次数和费用；
- 持仓数量；
- 每只基金贡献；
- 对应基准表现；
- 缺席日、未执行建议和模拟结果，但三者不得混入真实收益。

# 22. 月度报告

每个月结束后的第一次项目对话中生成上月报告，不要求用户必须在月末当天上线。

月度报告至少包含：

- 月初和月末资产；
- 单位净值和累计收益；
- 最大回撤；
- 总费用；
- 成功判断；
- 失败判断；
- 未执行建议和缺席日；
- 每只基金的10/15/30天表现；
- 绝对收益与对应基准；
- 资金效率和机会成本；
- 是否触发策略审计；
